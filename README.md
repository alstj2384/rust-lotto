## 러스트로 로또 구현
러스트로 우아한테크코스 프리코스 3주차 미션인 로또를 구현합니다.
기본적으로 우아한테크코스 요구사항을 따르며, 러스트의 장점을 살린 기능을 추가해보려고 합니다.

변경사항의 의도를 설명하기 위해 주석을 사용하였습니다!


## 기능 목록
### 입력

#### 로또 구입 금액

- [x] 양수를 입력받는다
- [x] [예외] 숫자가 아닌 값이 입력되면 예외 발생

#### 당첨 번호

- [x] 번호는 쉼표(,)를 기준으로 구분한다
- [x] 입력 사이 공백 허용 (예: 1, 2, 3)
- [x] 숫자 사이의 공백은 허용하지 않음 (예: "1 2")
- [x] [예외] 쉼표(,)와 숫자가 아닌 입력이 들어오면 예외 발생
- [x] [예외] 숫자 사이에 공백이 입력되면 예외 발생

#### 보너스 번호

- [x] 숫자를 입력받는다
- [x] [예외] 숫자가 아닌 값이 입력되면 예외 발생

### 출력

- [x] 발행한 로또 수량 출력
- [x] 발행한 로또 번호 출력 (오름차순 정렬)
- [x] 당첨 내역 출력
- [x] 수익률 출력
    - [x] 소수 둘째 자리에서 반올림
- [ ] 예외 메시지 출력

---

### 로또 발행

- [x] 금액을 입력받아 로또를 발행
- [ ] 로또 1장당 가격 단위(1000원) 검증
- [ ] 최대 구매 가능한 로또 개수는 1,000,000개
- [ ] [예외] 1장 미만 구매 시 예외 발생
- [ ] [예외] 1,000,000장 초과 구매 시 예외 발생

- [ ] 로또 번호의 숫자 범위는 1 ~ 45
- [x] 중복되지 않는 숫자 6개
- [ ] [예외] 범위를 벗어나는 경우 예외 발생
- [ ] [예외] 6자리가 아니면 예외 발생
- [ ] [예외] 번호가 중복되면 예외 발생

---

### 로또 추첨

- [ ] 중복되지 않는 숫자 6개 + 보너스 번호 1개
- [ ] [예외] 보너스 번호가 로또 번호 중복되면 예외 발생

---

### 당첨 여부 확인

다음 기준에 따른 당첨 결과를 판단한다:

- [x] **1등**: 6개 번호 일치 → 2,000,000,000원
- [x] **2등**: 5개 번호 + 보너스 번호 일치 → 30,000,000원
- [x] **3등**: 5개 번호 일치 → 1,500,000원
- [x] **4등**: 4개 번호 일치 → 50,000원
- [x] **5등**: 3개 번호 일치 → 5,000원

---

### 수익률 계산

- [x] 수익률 = (총 당첨 금액 / 총 구매 금액) × 100
- [x] 소수 둘째 자리에서 반올림

### 예외 처리 규칙

- [ ] 잘못된 값 입력 시 예외 발생
- [ ] 에러 메시지는 [ERROR]로 시작
- [ ] 예외 발생 시 해당 부분부터 입력을 다시 받음
- [ ] Exception이 아닌 명확한 예외 유형 사용


## 버전 네이밍 규칙 
```
v{major}.{minor}

major: 설계 변경, 새로운 기능 추가 등
minor: 기능 개선, 성능 최적화, 버그 수정
** 되는 쓰레기를 만드는 단계는 0.x 단계로 표시한다.
```
