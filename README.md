우아한테크코스 프리코스 미션인 `로또`를 `Rust`로 구현합니다.

해당 프로젝트는 `실행 환경의 여유 메모리`에 따라 `생성 가능한 최대 로또 개수`를 제시하고, `최대한 빠른 시간`에 생성해내는 것에 목표를 두었습니다.

# 프로젝트 특징
- 실행 환경에 따라 `최대로 생성할 수 있는 로또 개수`를 제시합니다.
- **최대 4개의 Thread**를 사용하여 **실행 시간을 단축**시킬 수 있습니다.
- **최대 2,000억원(2억개)** 까지 로또를 구매할 수 있습니다.
- **구매 로또 번호 출력 기능을 삭제**했습니다.
    - 주제 특성 상 매우 많은 로또가 생성되어야 하는데, 너무 많은 출력은 사용성을 저해한다고 판단했습니다.

### 설명을 위한 주석을 사용했습니다.
`Rust` 이해가 부족한 상황에서 짧은 시간에 구현한 프로젝트였기에, 제 3자 입장에서 코드 흐름을 한 눈에 파악하기 어려울 수 있다고 판단했습니다. 따라서 주석을 사용하여 프로그램 이해를 도왔습니다.


# 실행 방법
해당 프로젝트는 시스템 메모리 크기에 따라 생성 가능 로또 개수가 변화됩니다.

따라서, **Docker의 `--memory` 옵션을 이용하여 컨테이너 메모리를 제한하며 최대 생성 개수의 변화를 관찰**해보는 것을 추천드립니다.

## Docker (권장)
```
// 8 MB 메모리
docker run --rm -it --memory="8mb" --memory-swap="8mb" --cpus="4" alstj2384/rust-lotto:latest

// 512MB 메모리
docker run --rm -it --memory="512mb" --memory-swap="512mb" --cpus="4" alstj2384/rust-lotto:latest

// 1GB 메모리
docker run --rm -it --memory="1g" --memory-swap="1g" --cpus="4" alstj2384/rust-lotto:latest

// 2GB 메모리
docker run --rm -it --memory="2g" --memory-swap="2g" --cpus="4" alstj2384/rust-lotto:latest
```

## 기능 목록
### 입력

#### 로또 구입 금액

- [x] 양수를 입력받는다
- [x] [예외] 숫자가 아닌 값이 입력되면 예외 발생

#### 당첨 번호

- [x] 번호는 쉼표(,)를 기준으로 구분한다
- [x] 입력 사이 공백 허용 (예: 1, 2, 3)
- [x] 숫자 사이의 공백은 허용하지 않음 (예: "1 2")
- [x] [예외] 쉼표(,)와 숫자가 아닌 입력이 들어오면 예외 발생
- [x] [예외] 숫자 사이에 공백이 입력되면 예외 발생

#### 보너스 번호

- [x] 숫자를 입력받는다
- [x] [예외] 숫자가 아닌 값이 입력되면 예외 발생

### 출력

- [x] 발행한 로또 수량 출력
- [x] 발행한 로또 번호 출력 (오름차순 정렬)
- [x] 당첨 내역 출력
- [x] 수익률 출력
    - [x] 소수 둘째 자리에서 반올림
- [x] 예외 메시지 출력

---

### 로또 발행

- [x] 금액을 입력받아 로또를 발행
- [x] 로또 1장당 가격 단위(1000원) 검증
- [x] 최대 구매 가능한 로또 개수는 200,000,000개
- [x] [예외] 1장 미만 구매 시 예외 발생
- [x] [예외] 200,000,000장 초과 구매 시 예외 발생

- [x] 로또 번호의 숫자 범위는 1 ~ 45
- [x] 중복되지 않는 숫자 6개
- [x] [예외] 범위를 벗어나는 경우 예외 발생
- [x] [예외] 6자리가 아니면 예외 발생
- [x] [예외] 번호가 중복되면 예외 발생

---

### 로또 추첨

- [x] 중복되지 않는 숫자 6개 + 보너스 번호 1개
- [x] [예외] 보너스 번호가 로또 번호 중복되면 예외 발생

---

### 당첨 여부 확인

다음 기준에 따른 당첨 결과를 판단한다:

- [x] **1등**: 6개 번호 일치 → 2,000,000,000원
- [x] **2등**: 5개 번호 + 보너스 번호 일치 → 30,000,000원
- [x] **3등**: 5개 번호 일치 → 1,500,000원
- [x] **4등**: 4개 번호 일치 → 50,000원
- [x] **5등**: 3개 번호 일치 → 5,000원

---

### 수익률 계산

- [x] 수익률 = (총 당첨 금액 / 총 구매 금액) × 100
- [x] 소수 둘째 자리에서 반올림

### 예외 처리 규칙

- [x] 잘못된 값 입력 시 예외 발생
- [x] 에러 메시지는 [ERROR]로 시작
- [x] 예외 발생 시 해당 부분부터 입력을 다시 받음
